# Командний проєкт: Calculator (Лабораторна робота №5)

Цей репозиторій створено для виконання лабораторної роботи на тему **«Командна розробка в Git»**. 

##  Мета проєкту
Отримання практичних навичок спільної роботи над програмним кодом: створення гілок, робота з пул-запитами (Pull Requests) та вирішення конфліктів при злитті.

## 🛠 Технічний опис
У рамках проєкту реалізується програма-калькулятор. Вся основна логіка та математичні операції інкапсульовані в єдиному класі `Calculator`. Кожен метод покривається базовими тестовими випадками для перевірки його працездатності.

##  Команда та розподіл завдань

* **Павло (Паша) — Team Lead**
  * Створення репозиторію та ініціалізація проєкту.
  * Розробка базової структури класу `Calculator` (оголошення методів).
  * Реалізація базових методів (додавання, віднімання).
  * Code Review та затвердження Pull Requests.

* **Дем'ян — Розробник**
  * Гілка розробки: `feature-demian`
  * Реалізація методів множення та ділення (з перевіркою ділення на нуль).
  * Написання тестових випадків для своїх методів.

* **Влад — Розробник**
  * Гілка розробки: `feature-vlad`
  * Реалізація додаткових методів (наприклад, зведення в ступінь та відсотки).
  * Написання тестових випадків для своїх методів.

##  Робочий процес (Git Workflow)
1. Основна стабільна версія коду знаходиться у гілці `main`.
2. Розробка нових методів ведеться виключно в ізольованих гілках `feature-<ім'я/прізвище>`.
3. Після завершення написання коду створюється Pull Request у гілку `main`.
4. Злиття (Merge) відбувається тільки після перевірки (Review) Team Lead-ом та відсутності конфліктів.
